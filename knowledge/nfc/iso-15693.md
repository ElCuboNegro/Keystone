# ISO 15693 — Vicinity Integrated Circuit Cards
## Technical Reference for NFC/RFID Agent

---

## What it is

ISO 15693 defines **vicinity cards** — contactless smart cards readable at distances up to 1.5m.
Frequency: **13.56 MHz**.
Common use: inventory/asset tracking, access control, library books, transit passes.
Distinct from ISO 14443 (proximity cards, max ~10cm, think Mifare/credit cards).

---

## Physical Layer (ISO 15693-2)

| Parameter | Value |
|-----------|-------|
| Frequency | 13.56 MHz |
| Modulation (VCD→VICC) | ASK 10% or 100% |
| Data rate (low) | 1.65 kbps (fc/8192) |
| Data rate (high) | 26.48 kbps (fc/512) |
| Subcarrier | 423.75 kHz (fc/32) |
| Encoding | 1-of-4 or 1-of-256 pulse position |
| Response modulation | Load modulation via subcarrier |

VCD = Vicinity Coupling Device (the reader)
VICC = Vicinity Integrated Circuit Card (the tag)

## Protocol Layer (ISO 15693-3)

### Request Frame Format
```
SOF | Flags (1B) | Command Code (1B) | Parameters | CRC16 | EOF
```

### Response Frame Format
```
SOF | Flags (1B) | [Error Code (1B)] | Data | CRC16 | EOF
```

### Request Flags Byte
```
Bit 0: Sub-carrier flag    (0=single, 1=dual)
Bit 1: Data rate flag      (0=low, 1=high)
Bit 2: Inventory flag      (0=non-inventory, 1=inventory)
Bit 3: Protocol extension  (0=no extension)
--- when inventory flag = 0 ---
Bit 4: Select flag         (1=only respond if selected)
Bit 5: Address flag        (1=request is addressed, UID present)
Bit 6: Option flag         (command-specific)
Bit 7: RFU
--- when inventory flag = 1 ---
Bit 4: AFI flag            (1=AFI field present)
Bit 5: Nb_slots flag       (0=16 slots, 1=1 slot)
Bit 6: Option flag
Bit 7: RFU
```

### Response Flags Byte
```
Bit 0: Error flag          (1=error occurred, error code follows)
Bit 1-7: RFU
```

## Command Set

### Mandatory Commands
| Code | Command | Description |
|------|---------|-------------|
| 0x01 | INVENTORY | Anti-collision: enumerate all cards in field |
| 0x02 | STAY_QUIET | Put addressed card into quiet state |

### Optional Commands (commonly implemented)
| Code | Command | Description |
|------|---------|-------------|
| 0x20 | READ_SINGLE_BLOCK | Read one memory block |
| 0x21 | WRITE_SINGLE_BLOCK | Write one memory block |
| 0x22 | LOCK_BLOCK | Permanently lock a block |
| 0x23 | READ_MULTIPLE_BLOCKS | Read N consecutive blocks |
| 0x24 | WRITE_MULTIPLE_BLOCKS | Write N consecutive blocks |
| 0x25 | SELECT | Select a specific card |
| 0x26 | RESET_TO_READY | Deselect, return to ready state |
| 0x27 | WRITE_AFI | Write Application Family Identifier |
| 0x28 | LOCK_AFI | Lock AFI permanently |
| 0x29 | WRITE_DSFID | Write Data Storage Format Identifier |
| 0x2A | LOCK_DSFID | Lock DSFID permanently |
| 0x2B | GET_SYSTEM_INFORMATION | Read card's capabilities and UID |
| 0x2C | GET_MULTIPLE_BLOCK_SECURITY_STATUS | Read lock status of blocks |

### Custom/Manufacturer Commands
| Code range | Usage |
|-----------|-------|
| 0xA0–0xDF | Custom commands (manufacturer-defined, use Manufacturer Code) |
| 0xE0–0xFF | Proprietary commands |

## INVENTORY Command Detail (Critical for anti-collision)

### Request
```
Flags | 0x01 | [AFI] | Mask_Length | [Mask_Value]
```
- **Mask_Length**: number of mask bits (0 = enumerate all)
- Supports 1-slot or 16-slot anti-collision

### Response (per card)
```
Flags | DSFID (1B) | UID (8B)
```
- **UID is 8 bytes, LSB first** (byte 0 = LSB)
- Byte 7 always = 0xE0 (ISO 15693 manufacturer prefix)
- Bytes 6 = IC manufacturer code (e.g., 0x04 = NXP, 0x07 = Texas Instruments)

### UID Example
```
UID bytes: E0 04 01 00 12 34 56 78
Meaning:   [ISO] [NXP] [IC type] [serial number LSB→MSB]
Full UID:  78:56:34:12:00:01:04:E0  (MSB first display)
```

---

## GET_SYSTEM_INFORMATION Response

```
Flags | Info_Flags (1B) | UID (8B) | [DSFID] | [AFI] | [Block info] | [IC_Reference]
```

Info_Flags bits:
- Bit 0: DSFID supported
- Bit 1: AFI supported
- Bit 2: VICC memory size present
- Bit 3: IC reference present

Memory size field (if present):
```
Byte 0: Number of blocks - 1  (e.g., 0x1B = 28 blocks → 0x1B+1 = 28)
Byte 1: Block size - 1        (e.g., 0x03 = 4 bytes per block)
```

## Over PC/SC (via WinSCard / SCardTransmit)

When using an ACR122U or similar reader, ISO 15693 commands are wrapped in pseudo-APDUs.

### Card select via PC/SC
The PC/SC layer handles card selection automatically when `SCardConnect` succeeds.
On the ACR122U, ISO 15693 cards appear under protocol `SCARD_PROTOCOL_T1` or raw.

### Reading UID via PC/SC pseudo-APDU
```
FF CA 00 00 00   → Get UID (returns card UID + SW 90 00)
```

### Reading a block via PC/SC
For NFC/vicinity cards on ACR122U:
```
FF B0 00 <block_number> <length>
```
Response: `<data> 90 00`

### Writing a block
```
FF D6 00 <block_number> <length> <data>
```

### Direct ISO 15693 command via ACR122U escape
```
FF 00 00 00 <Lc> <ISO15693_command_bytes>
```
Example — Read single block 0:
```
FF 00 00 00 07  02 20 00 [UID 8 bytes if addressed]
Flags=0x02 (high data rate, addressed if UID present)
Command=0x20 (READ_SINGLE_BLOCK)
```

## Timing Considerations (CRITICAL for the millisecond problem)

### RF field must stay on
ISO 15693 cards are **passive** — they have no battery. They are powered ENTIRELY by the RF field.
If the reader turns off the RF field even for 1ms, the card loses power and resets.

### PC/SC layer pitfalls
1. **`SCardDisconnect(SCARD_UNPOWER_CARD)`** — explicitly turns off RF field. Card becomes unreachable.
2. **`SCardDisconnect(SCARD_RESET_CARD)`** — resets the card. Causes card to restart.
3. **`SCardDisconnect(SCARD_LEAVE_CARD)`** — safest: leaves card powered and in current state.
4. **No transaction** — without `SCardBeginTransaction`, the PC/SC layer may deselect the card between calls.

### ACR122U auto-polling
The ACR122U polls for cards by default, cycling through:
ISO 14443A → ISO 14443B → ISO 15693 → back to start

During this cycle, the RF field may briefly drop between protocol switches.
To stop auto-polling and hold field steady:
```
SCardControl(hCard, IOCTL_CCID_ESCAPE,
    {0xD4, 0x60, 0x19, 0x17},  // InSetParameters: disable auto-poll
    4, response, sizeof(response), &returned)
```

Or turn field on/off manually:
```
RF ON:  FF 00 00 00 04  D4 06 01 01
RF OFF: FF 00 00 00 04  D4 06 01 00
```

### Recommended session pattern for stable reads
```
SCardEstablishContext(SCARD_SCOPE_USER, ...)
SCardConnect(..., SCARD_SHARE_SHARED, SCARD_PROTOCOL_T1, ...)
SCardBeginTransaction(hCard)
  // Do all reads/writes here
  SCardTransmit(...)  // read UID
  SCardTransmit(...)  // read blocks
SCardEndTransaction(hCard, SCARD_LEAVE_CARD)
SCardDisconnect(hCard, SCARD_LEAVE_CARD)
```

## Linux Implementation Notes

### pyscard (Python, cross-platform)
```python
from smartcard.System import readers
from smartcard.util import toHexString

r = readers()[0]
connection = r.createConnection()
connection.connect()

# Get UID
GET_UID = [0xFF, 0xCA, 0x00, 0x00, 0x00]
data, sw1, sw2 = connection.transmit(GET_UID)
print(f"UID: {toHexString(data)}, SW: {sw1:02X}{sw2:02X}")
```

### libnfc (C, direct USB, no pcscd needed)
```c
nfc_context *context;
nfc_init(&context);

nfc_device *device = nfc_open(context, NULL);
nfc_initiator_init(device);

nfc_modulation modulation = {
    .nmt = NMT_ISO14443B,   // or NMT_ISO15693 for vicinity
    .nbr = NBR_106,
};

nfc_target target;
if (nfc_initiator_select_passive_target(device, modulation, NULL, 0, &target) > 0) {
    // card found, target.nti.nai.abtUid has the UID
}
```
